# Spidercrawl: Implementation Plan

Spidercrawl is an AI-native web intelligence engine designed to surpass existing tools like Firecrawl by providing multimodal extraction, goal-oriented crawling, and self-healing extraction capabilities.

## 1. Project Vision
To create a high-performance, intelligent web crawler that transforms the unstructured web into a structured, semantic knowledge base optimized for LLMs.

---

## 2. Competitive Analysis: Firecrawl vs. Spidercrawl

| Feature | Firecrawl | Spidercrawl (Ours) |
| :--- | :--- | :--- |
| **Rendering** | Playwright/L5 | Adaptive Rendering (Cheerio for speed, Playwright for JS) |
| **Extraction** | Markdown/JSON | **Multimodal Markdown** (Text + Image/Chart Descriptions) |
| **Crawling** | Breadth-First (Dumb) | **Goal-Oriented** (AI prioritizes relevant links) |
| **Resilience** | Standard Proxy/Stealth | **Self-Healing** (AI re-maps structure on layout change) |
| **Output** | Flat Files | **Knowledge Graph** (Semantic relationships preserved) |
| **Hosting** | SaaS First | **Hybrid** (Local-First core with SaaS scale hooks) |

## 3. Detailed Architecture

### A. Core Modules
1.  **API Gateway (Node.js/Express/Fastify)**
    *   RESTful interface for triggering crawls and scrapes.
    *   Authentication and Rate Limiting.
    *   Webhook management for async job completion.

2.  **Orchestrator & Queue (BullMQ + Redis)**
    *   Manages the lifecycle of a crawl job.
    *   Handles retries, concurrency limits, and state persistence.
    *   Prioritizes "Goal-Oriented" URLs.

3.  **The Stealth Scraper (Python/Playwright)**
    *   Uses `Crawl4AI` and `Playwright Stealth` for maximum bypass.
    *   Handles session management and automated interactions (scroll/click).
    *   Extracts raw HTML, screenshots, and accessibility trees.

4.  **AI Processing Pipeline**
    *   **Cleaner:** Strips boilerplate (nav, footer, ads).
    *   **Vision Engine:** Analyzes images/charts via Gemini/GPT-4o Vision.
    *   **Structured Extractor:** Maps content to user-defined (or AI-suggested) JSON schemas.

5.  **Data Storage & Export**
    *   **PostgreSQL/Supabase:** For job metadata and extracted text.
    *   **Vector DB (Optional):** For immediate RAG readiness.
    *   **Export Formats:** Markdown, JSON, CSV, JSON-LD (Knowledge Graph).

## 4. Feature Breakdown & Logic

### Goal-Oriented Crawling
*   **Input:** URL + "Find all technical documentation for the API."
*   **Logic:** For every discovered link, the AI scores it (0-1) based on the goal. The queue is sorted by this score.

### Multimodal Extraction
*   **Logic:**
    1. Detect `<img>` or `<canvas>` tags.
    2. Filter for significant images (ignore icons/trackers).
    3. Send image + page context to Vision LLM.
    4. Replace image tag in Markdown with `![Description generated by Spidercrawl Vision](...)`.

### Self-Healing Extraction
*   **Logic:** If the user requests a "Price" field and it's missing, the engine performs a "Deep Inspect" where it renders the page with full interaction (clicking buttons that might reveal price) and uses AI to find the data in the DOM tree.

## 5. Technology Stack
*   **Backend:** Node.js (TypeScript)
*   **Workers:** Python 3.11+
*   **Browser:** Playwright (Chromium)
*   **Database:** Redis (Queue) + PostgreSQL (Data)
*   **AI:** OpenAI API / Anthropic / Google Gemini (Vision/Extraction)
*   **Orchestration:** Docker Compose / Kubernetes
