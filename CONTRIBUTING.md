# Contributing to Spidercrawl

First off, thank you for considering contributing to Spidercrawl! It's people like you that make Spidercrawl such a great tool.

## How Can I Contribute?

### Reporting Bugs
* **Check the Issues**: Before creating a new issue, please check if the bug has already been reported.
* **Use the Template**: When reporting a bug, use the [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md) template.
* **Be Specific**: Include as much detail as possible, such as your environment, steps to reproduce, and expected vs actual behavior.

### Suggesting Enhancements
* **Check the Issues**: See if someone else has already suggested the enhancement.
* **Describe the Use Case**: Explain why this enhancement would be useful to most Spidercrawl users.

### Pull Requests
1. **Fork the Repo**: Create your own fork of the project.
2. **Create a Branch**: `git checkout -b feature/my-new-feature`
3. **Run Tests**: Ensure all tests pass with `npm test`.
4. **Commit Changes**: Use clear and descriptive commit messages (following Conventional Commits is preferred).
5. **Push and Open PR**: Push to your fork and submit a PR to the `main` branch.

## Development Setup

See the [README.md](README.md#quick-start-local-development) for detailed instructions on setting up your local environment.

## Style Guide
* Follow the existing code style (ESLint and Prettier are configured).
* Use TypeScript for all new dashboard and API code.
* Write unit tests for new features.
